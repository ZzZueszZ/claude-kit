# Oracle Database

Các convention và best practices khi làm việc với Oracle Database.

---

## 1. Oracle Data Types

### Bảng Data Types thường dùng

| Category | Data Type | Mô tả | Khi nào dùng |
|----------|----------|--------|-------------|
| **String** | `VARCHAR2(n)` | Chuỗi variable-length (max 4000) | Hầu hết string columns |
| | `NVARCHAR2(n)` | Unicode variable-length | Dữ liệu đa ngôn ngữ |
| | `CHAR(n)` | Fixed-length | Code cố định (VD: `CHAR(3)` cho country code) |
| | `CLOB` | Character LOB (>4000 chars) | Text rất dài |
| **Number** | `NUMBER(p,s)` | Số với precision, scale | Tiền tệ, số lượng |
| | `NUMBER` | Số không giới hạn precision | ID (khi không dùng IDENTITY) |
| | `BINARY_FLOAT` | 32-bit floating | Scientific data (KHÔNG dùng cho tiền) |
| **Date/Time** | `DATE` | Ngày + giờ (chính xác đến giây) | `birth_date`, `order_date` |
| | `TIMESTAMP` | Ngày + giờ + fractional seconds | `created_at`, `updated_at` |
| | `TIMESTAMP WITH TIME ZONE` | Timestamp + timezone | Ứng dụng multi-timezone |
| | `INTERVAL` | Khoảng thời gian | Duration, periods |
| **Binary** | `BLOB` | Binary LOB | Files, images |
| | `RAW(n)` | Raw binary (max 2000) | UUID, hash values |
| **JSON** | `CLOB` + IS JSON | JSON text | Oracle 12c+ |
| | `JSON` | Native JSON | Oracle 21c+ |

### Quy tắc bắt buộc
- **LUÔN** dùng `VARCHAR2`, **KHÔNG** dùng `VARCHAR` (behavior khác nhau trong Oracle)
- **LUÔN** specify precision cho `NUMBER` khi lưu tiền tệ: `NUMBER(15,2)`
- **KHÔNG** dùng `LONG` — deprecated, dùng `CLOB` thay thế
- Dùng `DATE` thay `TIMESTAMP` khi không cần fractional seconds
- **KHÔNG** dùng `CHAR` trừ fixed-length codes (waste storage)

### ✅ Đúng
```sql
CREATE TABLE employees (
    id              NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    employee_code   CHAR(8)           NOT NULL,     -- Fixed: 'EMP00001'
    full_name       VARCHAR2(200)     NOT NULL,
    email           VARCHAR2(255)     NOT NULL,
    salary          NUMBER(15,2)      NOT NULL,
    birth_date      DATE,
    hired_at        TIMESTAMP         DEFAULT SYSTIMESTAMP NOT NULL,
    profile_data    CLOB              CHECK (profile_data IS JSON),
    avatar          BLOB,
    CONSTRAINT uq_employees_code UNIQUE (employee_code),
    CONSTRAINT uq_employees_email UNIQUE (email),
    CONSTRAINT chk_employees_salary CHECK (salary > 0)
);
```

## 2. Sequences & Identity Columns

### Oracle 12c+ — Identity Column (Recommended)
```sql
-- GENERATED BY DEFAULT: Cho phép insert manual ID
CREATE TABLE users (
    id   NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name VARCHAR2(100) NOT NULL
);

-- GENERATED ALWAYS: Luôn auto-generate (không cho insert manual)
CREATE TABLE audit_logs (
    id      NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    action  VARCHAR2(50) NOT NULL,
    details CLOB
);
```

### Sequence thủ công (cho Oracle 11g hoặc custom behavior)
```sql
-- Tạo sequence
CREATE SEQUENCE seq_orders
    START WITH 1
    INCREMENT BY 1
    NOCACHE       -- Dùng CACHE 20 cho high-throughput
    NOCYCLE;

-- Sử dụng trong INSERT
INSERT INTO orders (id, user_id, total_amount)
VALUES (seq_orders.NEXTVAL, :user_id, :total_amount);

-- Lấy giá trị hiện tại (sau NEXTVAL)
SELECT seq_orders.CURRVAL FROM DUAL;
```

### Sequence + Trigger (Legacy pattern — Oracle 11g)
```sql
-- ⚠️ Chỉ dùng khi không có IDENTITY (Oracle 11g)
CREATE OR REPLACE TRIGGER trg_users_bi
BEFORE INSERT ON users
FOR EACH ROW
WHEN (NEW.id IS NULL)
BEGIN
    :NEW.id := seq_users.NEXTVAL;
END;
/
```

### Quy tắc
- **ƯU TIÊN** `GENERATED BY DEFAULT AS IDENTITY` (Oracle 12c+)
- Dùng `CACHE` cho sequences high-throughput: `CREATE SEQUENCE ... CACHE 100`
- **KHÔNG** dùng trigger để set ID nếu dùng Oracle 12c+
- Dùng `NOCYCLE` trừ khi có use case cụ thể cho cycling

## 3. PL/SQL Cơ bản

### Stored Procedure
```sql
CREATE OR REPLACE PROCEDURE sp_deactivate_inactive_users (
    p_days_inactive IN NUMBER DEFAULT 90,
    p_count         OUT NUMBER
) AS
BEGIN
    UPDATE users
    SET    status     = 'INACTIVE',
           updated_at = SYSTIMESTAMP
    WHERE  status = 'ACTIVE'
    AND    last_login_at < SYSTIMESTAMP - p_days_inactive;

    p_count := SQL%ROWCOUNT;
    COMMIT;

EXCEPTION
    WHEN OTHERS THEN
        ROLLBACK;
        RAISE;
END sp_deactivate_inactive_users;
/
```

### Function
```sql
CREATE OR REPLACE FUNCTION fn_calculate_order_total (
    p_order_id IN NUMBER
) RETURN NUMBER
IS
    v_total NUMBER(15,2);
BEGIN
    SELECT NVL(SUM(quantity * unit_price), 0)
    INTO   v_total
    FROM   order_items
    WHERE  order_id = p_order_id;

    RETURN v_total;

EXCEPTION
    WHEN NO_DATA_FOUND THEN
        RETURN 0;
END fn_calculate_order_total;
/
```

### Package
```sql
-- Package Specification (Header)
CREATE OR REPLACE PACKAGE pkg_user_management AS
    PROCEDURE create_user(
        p_name     IN VARCHAR2,
        p_email    IN VARCHAR2,
        p_user_id  OUT NUMBER
    );

    FUNCTION get_user_count RETURN NUMBER;

    PROCEDURE deactivate_user(p_user_id IN NUMBER);
END pkg_user_management;
/

-- Package Body (Implementation)
CREATE OR REPLACE PACKAGE BODY pkg_user_management AS

    PROCEDURE create_user(
        p_name     IN VARCHAR2,
        p_email    IN VARCHAR2,
        p_user_id  OUT NUMBER
    ) IS
    BEGIN
        INSERT INTO users (name, email, status, created_at)
        VALUES (p_name, p_email, 'ACTIVE', SYSTIMESTAMP)
        RETURNING id INTO p_user_id;
    END create_user;

    FUNCTION get_user_count RETURN NUMBER IS
        v_count NUMBER;
    BEGIN
        SELECT COUNT(*) INTO v_count FROM users WHERE status = 'ACTIVE';
        RETURN v_count;
    END get_user_count;

    PROCEDURE deactivate_user(p_user_id IN NUMBER) IS
    BEGIN
        UPDATE users
        SET    status = 'INACTIVE', updated_at = SYSTIMESTAMP
        WHERE  id = p_user_id;

        IF SQL%ROWCOUNT = 0 THEN
            RAISE_APPLICATION_ERROR(-20001, 'User not found: ' || p_user_id);
        END IF;
    END deactivate_user;

END pkg_user_management;
/
```

### Quy tắc PL/SQL
- **LUÔN** có `EXCEPTION` block cho stored procedures/functions
- Dùng `RAISE_APPLICATION_ERROR(-20xxx, 'message')` cho custom errors
- **KHÔNG** dùng `DBMS_OUTPUT.PUT_LINE` trong production code
- Dùng **Packages** để nhóm related procedures/functions
- Naming: `sp_` cho procedures, `fn_` cho functions, `pkg_` cho packages, `trg_` cho triggers

## 4. Partitioning

### Khi nào partition
- Bảng > **10 triệu rows** hoặc > **2GB**
- Queries luôn có filter trên partition key
- Cần purge data định kỳ (DROP PARTITION nhanh hơn DELETE)

### Range Partitioning (phổ biến nhất)
```sql
CREATE TABLE orders (
    id           NUMBER GENERATED BY DEFAULT AS IDENTITY,
    user_id      NUMBER        NOT NULL,
    order_date   DATE          NOT NULL,
    total_amount NUMBER(15,2)  NOT NULL,
    status       VARCHAR2(20)  NOT NULL
)
PARTITION BY RANGE (order_date) (
    PARTITION p_2024_q1 VALUES LESS THAN (DATE '2024-04-01'),
    PARTITION p_2024_q2 VALUES LESS THAN (DATE '2024-07-01'),
    PARTITION p_2024_q3 VALUES LESS THAN (DATE '2024-10-01'),
    PARTITION p_2024_q4 VALUES LESS THAN (DATE '2025-01-01'),
    PARTITION p_max     VALUES LESS THAN (MAXVALUE)
);
```

### List Partitioning
```sql
CREATE TABLE transactions (
    id          NUMBER GENERATED BY DEFAULT AS IDENTITY,
    region      VARCHAR2(20) NOT NULL,
    amount      NUMBER(15,2) NOT NULL,
    created_at  TIMESTAMP    NOT NULL
)
PARTITION BY LIST (region) (
    PARTITION p_north  VALUES ('HA_NOI', 'HAI_PHONG', 'QUANG_NINH'),
    PARTITION p_central VALUES ('DA_NANG', 'HUE', 'NHA_TRANG'),
    PARTITION p_south  VALUES ('HO_CHI_MINH', 'CAN_THO', 'BINH_DUONG'),
    PARTITION p_other  VALUES (DEFAULT)
);
```

### Hash Partitioning
```sql
-- Phân bố đều data, dùng khi không có natural range/list key
CREATE TABLE session_data (
    session_id  VARCHAR2(100) PRIMARY KEY,
    user_id     NUMBER NOT NULL,
    data        CLOB
)
PARTITION BY HASH (session_id)
PARTITIONS 8;
```

### Partition Maintenance
```sql
-- Thêm partition mới
ALTER TABLE orders ADD PARTITION p_2025_q1
    VALUES LESS THAN (DATE '2025-04-01');

-- Drop partition cũ (purge data nhanh)
ALTER TABLE orders DROP PARTITION p_2024_q1;

-- Split MAXVALUE partition
ALTER TABLE orders SPLIT PARTITION p_max AT (DATE '2025-04-01')
    INTO (PARTITION p_2025_q1, PARTITION p_max);
```

## 5. Performance Hints & Tuning

### Oracle Hints thường dùng

| Hint | Mục đích | Khi nào dùng |
|------|---------|-------------|
| `/*+ INDEX(t idx_name) */` | Force dùng index cụ thể | Optimizer chọn sai index |
| `/*+ FULL(t) */` | Force full table scan | Query lấy nhiều % data |
| `/*+ PARALLEL(t, 4) */` | Parallel execution | Batch processing, large scans |
| `/*+ LEADING(t1 t2) */` | Chỉ định join order | Optimizer chọn sai join order |
| `/*+ USE_NL(t) */` | Nested Loop join | Small driving table |
| `/*+ USE_HASH(t) */` | Hash join | Large unsorted datasets |
| `/*+ NO_MERGE(v) */` | Không merge subquery/view | Complex subqueries |
| `/*+ FIRST_ROWS(n) */` | Optimize cho n rows đầu | Pagination queries |

### ⚠️ Quy tắc dùng Hints
- **CHỈ** dùng hints khi optimizer chọn plan kém
- **LUÔN** xác minh bằng Execution Plan trước và sau khi thêm hint
- **DOCUMENT** lý do dùng hint trong comment
- Hints là **giải pháp cuối cùng**, ưu tiên fix root cause (statistics, index)

### Gather Statistics
```sql
-- Gather table statistics (quan trọng cho optimizer)
BEGIN
    DBMS_STATS.GATHER_TABLE_STATS(
        ownname => 'SCHEMA_NAME',
        tabname => 'ORDERS',
        estimate_percent => DBMS_STATS.AUTO_SAMPLE_SIZE,
        method_opt => 'FOR ALL COLUMNS SIZE AUTO',
        cascade => TRUE  -- Include indexes
    );
END;
/

-- Gather stale statistics cho toàn bộ schema
BEGIN
    DBMS_STATS.GATHER_SCHEMA_STATS(
        ownname => 'SCHEMA_NAME',
        options => 'GATHER STALE'
    );
END;
/
```

### Bulk Operations
```sql
-- ✅ FORALL — bulk DML (nhanh hơn FOR loop nhiều lần)
DECLARE
    TYPE t_ids IS TABLE OF NUMBER;
    v_ids t_ids := t_ids(1, 2, 3, 4, 5);
BEGIN
    FORALL i IN 1..v_ids.COUNT
        UPDATE orders
        SET    status = 'ARCHIVED'
        WHERE  id = v_ids(i);
    COMMIT;
END;
/

-- ✅ BULK COLLECT — fetch nhiều rows cùng lúc
DECLARE
    TYPE t_users IS TABLE OF users%ROWTYPE;
    v_users t_users;
BEGIN
    SELECT * BULK COLLECT INTO v_users
    FROM   users
    WHERE  status = 'ACTIVE';
    -- Process v_users...
END;
/

-- ✅ BULK COLLECT với LIMIT (tránh memory overflow)
DECLARE
    CURSOR c_orders IS SELECT * FROM orders WHERE status = 'PENDING';
    TYPE t_orders IS TABLE OF orders%ROWTYPE;
    v_orders t_orders;
BEGIN
    OPEN c_orders;
    LOOP
        FETCH c_orders BULK COLLECT INTO v_orders LIMIT 1000;
        EXIT WHEN v_orders.COUNT = 0;
        -- Process batch of 1000...
    END LOOP;
    CLOSE c_orders;
END;
/
```

## 6. Tablespace & Storage

### Tablespace Naming Convention
```sql
-- Tablespace cho data
CREATE TABLESPACE ts_app_data
    DATAFILE '/u01/oradata/mydb/ts_app_data01.dbf'
    SIZE 1G AUTOEXTEND ON NEXT 256M MAXSIZE 10G;

-- Tablespace cho indexes
CREATE TABLESPACE ts_app_index
    DATAFILE '/u01/oradata/mydb/ts_app_index01.dbf'
    SIZE 512M AUTOEXTEND ON NEXT 128M MAXSIZE 5G;

-- Specify tablespace khi tạo table/index
CREATE TABLE orders (...)
    TABLESPACE ts_app_data;

CREATE INDEX idx_orders_user_id ON orders(user_id)
    TABLESPACE ts_app_index;
```

### Quy tắc
- **TÁCH** tablespace cho data và index
- **LUÔN** enable `AUTOEXTEND` với `MAXSIZE`
- **KHÔNG** dùng `SYSTEM` hoặc `SYSAUX` tablespace cho application data

## 7. Oracle + Spring Boot Integration

### application.yml
```yaml
spring:
  datasource:
    url: jdbc:oracle:thin:@${DB_HOST:localhost}:${DB_PORT:1521}/${DB_SERVICE:XEPDB1}
    username: ${DB_USERNAME}
    password: ${DB_PASSWORD}
    driver-class-name: oracle.jdbc.OracleDriver
    hikari:
      maximum-pool-size: ${DB_POOL_SIZE:10}
      minimum-idle: ${DB_POOL_MIN:5}
      connection-timeout: 30000
      idle-timeout: 600000
      max-lifetime: 1800000
  jpa:
    database-platform: org.hibernate.dialect.OracleDialect
    hibernate:
      ddl-auto: validate  # LUÔN validate trong production
    properties:
      hibernate:
        default_schema: ${DB_SCHEMA}
        format_sql: true
```

### Oracle-specific JPA Mapping
```java
@Entity
@Table(name = "users")
@Getter @Setter
@NoArgsConstructor(access = AccessLevel.PROTECTED)
public class User extends BaseEntity {

    // Oracle IDENTITY column
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    // Oracle VARCHAR2 → String
    @Column(name = "full_name", length = 200, nullable = false)
    private String fullName;

    // Oracle NUMBER(15,2) → BigDecimal
    @Column(name = "salary", precision = 15, scale = 2)
    private BigDecimal salary;

    // Oracle DATE → LocalDate
    @Column(name = "birth_date")
    private LocalDate birthDate;

    // Oracle TIMESTAMP → LocalDateTime
    @Column(name = "created_at", nullable = false, updatable = false)
    private LocalDateTime createdAt;

    // Oracle CLOB → String (with @Lob)
    @Lob
    @Column(name = "description")
    private String description;
}
```

## Checklist khi làm việc với Oracle

- [ ] Dùng `VARCHAR2` (không dùng `VARCHAR`)
- [ ] `NUMBER` có precision/scale rõ ràng cho tiền tệ
- [ ] Dùng `IDENTITY` column (Oracle 12c+) thay sequence + trigger
- [ ] Index trên FK columns (Oracle **KHÔNG** tự tạo)
- [ ] Statistics up-to-date (`DBMS_STATS`)
- [ ] Tách tablespace cho data và index
- [ ] PL/SQL có `EXCEPTION` block
- [ ] Bulk operations dùng `FORALL` / `BULK COLLECT`
- [ ] Hints chỉ dùng khi có evidence từ Execution Plan
- [ ] HikariCP config phù hợp cho Oracle connection
