## 1. Oracle Data Types

### Mapping Java ↔ Oracle

| Java Type | Oracle Type | Ghi chú |
|-----------|-------------|---------|
| `Long` | `NUMBER` hoặc `NUMBER(19)` | Primary key, FK |
| `Integer` | `NUMBER(10)` | Số nguyên thường |
| `String` (ngắn) | `VARCHAR2(n)` | Tối đa 4000 bytes |
| `String` (dài) | `CLOB` | Text lớn (body, content) |
| `Boolean` / `boolean` | `NUMBER(1)` | 0 = false, 1 = true |
| `BigDecimal` | `NUMBER(p,s)` | Tiền tệ: `NUMBER(19,4)` |
| `LocalDateTime` | `TIMESTAMP` | Thời gian có fractional seconds |
| `LocalDate` | `DATE` | Chỉ ngày, không giờ |
| `Enum` (STRING) | `VARCHAR2(n)` | `@Enumerated(EnumType.STRING)` |
| `Enum` (ORDINAL) | `NUMBER(2)` | `@Enumerated(EnumType.ORDINAL)` |
| `JSON Object` | `CLOB` + `@JdbcTypeCode` | Lưu JSON dạng text |

### Quy tắc bắt buộc
- **LUÔN** dùng `VARCHAR2` — KHÔNG dùng `VARCHAR` (Oracle khác biệt)
- **LUÔN** chỉ định length cho `VARCHAR2`: `VARCHAR2(100)`, KHÔNG để trống
- **LUÔN** dùng `NUMBER(1)` cho boolean — Oracle KHÔNG có kiểu `BOOLEAN` trong table
- **KHÔNG** dùng `CHAR` trừ khi dữ liệu luôn cố định chiều dài (VD: mã quốc gia 2 ký tự)
- Dùng `TIMESTAMP` thay `DATE` khi cần lưu giờ/phút/giây chính xác

## 2. DDL — Data Definition Language

### Quy tắc đặt tên
- Tên bảng: `snake_case`, số nhiều → `business_postbox`, `citizen_messages`
- Tên cột: `snake_case` → `created_at`, `owner_id`
- Tên constraint: `{prefix}_{table}_{column}` → `pk_users_id`, `fk_orders_user_id`
- Tên index: `idx_{table}_{column}` → `idx_users_email`
- Tên sequence: `seq_{table}` → `seq_users`
- **LUÔN** quote tên bằng dấu `"` nếu cần preserve case

### Tạo bảng chuẩn

```sql
-- Tạo sequence cho auto-increment
CREATE SEQUENCE "seq_users" START WITH 1 INCREMENT BY 1 NOCACHE;

-- Tạo bảng
CREATE TABLE "users" (
    "id"                  NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    "username"            VARCHAR2(50)    NOT NULL,
    "email"               VARCHAR2(255)   NOT NULL,
    "password"            VARCHAR2(255)   NOT NULL,
    "full_name"           VARCHAR2(100)   DEFAULT NULL,
    "status"              VARCHAR2(20)    DEFAULT 'ACTIVE' NOT NULL,
    "deleted"             NUMBER(1)       DEFAULT 0 NOT NULL,
    -- Audit fields (PropertyV2Auditable)
    "property_id"         NUMBER          DEFAULT NULL,
    "created_at"          TIMESTAMP       DEFAULT CURRENT_TIMESTAMP,
    "updated_at"          TIMESTAMP       DEFAULT CURRENT_TIMESTAMP,
    "created_by"          NUMBER          DEFAULT 0,
    "updated_by"          NUMBER          DEFAULT 0,
    "tid"                 VARCHAR2(100)   DEFAULT '',
    -- Constraints
    CONSTRAINT "uk_users_email" UNIQUE ("email"),
    CONSTRAINT "uk_users_username" UNIQUE ("username"),
    CONSTRAINT "ck_users_status" CHECK ("status" IN ('ACTIVE', 'INACTIVE', 'SUSPENDED'))
);

-- Comments
COMMENT ON TABLE "users" IS 'Bảng quản lý người dùng hệ thống';
COMMENT ON COLUMN "users"."status" IS 'Trạng thái: ACTIVE, INACTIVE, SUSPENDED';
COMMENT ON COLUMN "users"."deleted" IS 'Soft delete: 0 = active, 1 = deleted';

-- Index
CREATE INDEX "idx_users_status" ON "users" ("status");
CREATE INDEX "idx_users_property_id" ON "users" ("property_id");
```

### Identity Column (Oracle 12c+)

```sql
-- Dùng GENERATED ALWAYS AS IDENTITY (khuyến nghị)
"id" NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY

-- Hoặc GENERATED BY DEFAULT nếu cần insert thủ công
"id" NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY

-- Với tuỳ chọn cache cho performance
"id" NUMBER GENERATED ALWAYS AS IDENTITY (START WITH 1 INCREMENT BY 1 CACHE 20) PRIMARY KEY
```

### Alter Table

```sql
-- Thêm cột
ALTER TABLE "users" ADD "phone_number" VARCHAR2(20) DEFAULT NULL;

-- Sửa cột
ALTER TABLE "users" MODIFY "full_name" VARCHAR2(200);

-- Xoá cột
ALTER TABLE "users" DROP COLUMN "phone_number";

-- Thêm constraint
ALTER TABLE "users" ADD CONSTRAINT "ck_users_email_format" 
    CHECK (REGEXP_LIKE("email", '^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$'));

-- Rename cột (Oracle 9i+)
ALTER TABLE "users" RENAME COLUMN "full_name" TO "display_name";
```

## 3. DML — Data Manipulation Language

### INSERT

```sql
-- Single insert
INSERT INTO "users" ("username", "email", "password", "status")
VALUES ('john', 'john@example.com', 'hashed_password', 'ACTIVE');

-- Insert with returning (lấy ID vừa insert)
INSERT INTO "users" ("username", "email", "password")
VALUES ('jane', 'jane@example.com', 'hashed_password')
RETURNING "id" INTO :new_id;

-- Multi-row insert (Oracle)
INSERT ALL
    INTO "users" ("username", "email", "password") VALUES ('user1', 'user1@mail.com', 'pwd1')
    INTO "users" ("username", "email", "password") VALUES ('user2', 'user2@mail.com', 'pwd2')
    INTO "users" ("username", "email", "password") VALUES ('user3', 'user3@mail.com', 'pwd3')
SELECT 1 FROM DUAL;
```

### SELECT

```sql
-- Phân trang Oracle (12c+ — FETCH FIRST)
SELECT "id", "username", "email", "status"
FROM "users"
WHERE "status" = 'ACTIVE' AND "deleted" = 0
ORDER BY "created_at" DESC
OFFSET 0 ROWS FETCH NEXT 20 ROWS ONLY;

-- Phân trang Oracle (trước 12c — ROWNUM)
SELECT * FROM (
    SELECT t.*, ROWNUM rn FROM (
        SELECT "id", "username", "email"
        FROM "users"
        WHERE "status" = 'ACTIVE'
        ORDER BY "created_at" DESC
    ) t WHERE ROWNUM <= 20
) WHERE rn > 0;

-- COUNT với điều kiện
SELECT COUNT(*) AS total,
       COUNT(CASE WHEN "status" = 'ACTIVE' THEN 1 END) AS active_count,
       COUNT(CASE WHEN "status" = 'INACTIVE' THEN 1 END) AS inactive_count
FROM "users"
WHERE "deleted" = 0;
```

### UPDATE

```sql
-- Update thông thường
UPDATE "users"
SET "status" = 'INACTIVE',
    "updated_at" = CURRENT_TIMESTAMP,
    "updated_by" = :userId
WHERE "id" = :id AND "deleted" = 0;

-- Update với subquery
UPDATE "citizen_messages"
SET "status" = 'SEEN', "seen_at" = CURRENT_TIMESTAMP
WHERE "postbox_id" = :postboxId
  AND "status" = 'UNSEEN'
  AND "deleted" = 0;
```

### DELETE / Soft Delete

```sql
-- ❌ KHÔNG hard delete trong production
DELETE FROM "users" WHERE "id" = :id;

-- ✅ LUÔN soft delete
UPDATE "users"
SET "deleted" = 1,
    "updated_at" = CURRENT_TIMESTAMP,
    "updated_by" = :userId
WHERE "id" = :id AND "deleted" = 0;
```

## 4. Constraints

### Các loại constraint

```sql
-- Primary Key
CONSTRAINT "pk_users" PRIMARY KEY ("id")

-- Foreign Key
CONSTRAINT "fk_orders_user_id" FOREIGN KEY ("user_id") 
    REFERENCES "users" ("id") ON DELETE CASCADE

-- Unique
CONSTRAINT "uk_users_email" UNIQUE ("email")

-- Unique composite
CONSTRAINT "uk_label_msgs" UNIQUE ("label_id", "message_id")

-- Check
CONSTRAINT "ck_users_status" CHECK ("status" IN ('ACTIVE', 'INACTIVE'))

-- Not Null (inline)
"email" VARCHAR2(255) NOT NULL
```

### Quy tắc FK
- **ƯU TIÊN** `ON DELETE SET NULL` cho soft-reference
- Dùng `ON DELETE CASCADE` chỉ cho quan hệ parent-child chặt
- **KHÔNG** dùng `ON DELETE CASCADE` cho quan hệ loose coupling
- **LUÔN** tạo index cho FK columns → tránh full table lock khi parent bị delete/update

```sql
-- LUÔN tạo index cho FK
CREATE INDEX "idx_orders_user_id" ON "orders" ("user_id");
```

## 5. Sequences & Auto-Increment

### Oracle Identity Column (khuyến nghị cho Oracle 12c+)

```sql
-- Không cần tạo sequence riêng
CREATE TABLE "labels" (
    "id" NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    ...
);
```

### Oracle Sequence (cho Oracle trước 12c hoặc cần kiểm soát chi tiết)

```sql
-- Tạo sequence
CREATE SEQUENCE "seq_labels"
    START WITH 1
    INCREMENT BY 1
    NOCACHE
    NOCYCLE;

-- Dùng trong INSERT
INSERT INTO "labels" ("id", "name") VALUES ("seq_labels".NEXTVAL, 'Urgent');

-- Lấy giá trị hiện tại
SELECT "seq_labels".CURRVAL FROM DUAL;
```

### JPA mapping với Sequence

```java
@Entity
@Table(name = "labels")
public class LabelEntity extends PropertyV2Auditable {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    // ...
}

// Hoặc với sequence cụ thể
@Entity
public class LabelEntity extends PropertyV2Auditable {

    @Id
    @GeneratedValue(strategy = GenerationType.SEQUENCE, generator = "label_seq")
    @SequenceGenerator(name = "label_seq", sequenceName = "seq_labels", allocationSize = 1)
    private Long id;
}
```

## 6. Oracle-Specific Functions

### Xử lý NULL

```sql
-- NVL — thay NULL bằng giá trị mặc định (tương đương COALESCE cho 2 tham số)
SELECT NVL("full_name", 'N/A') FROM "users";

-- NVL2 — if not null return A, else return B
SELECT NVL2("phone_number", 'Có SĐT', 'Chưa có SĐT') FROM "users";

-- COALESCE — lấy giá trị đầu tiên không NULL
SELECT COALESCE("display_name", "full_name", "username") AS name FROM "users";

-- NULLIF — trả về NULL nếu 2 giá trị bằng nhau
SELECT NULLIF("status", 'UNKNOWN') FROM "users";
```

### Xử lý chuỗi

```sql
-- Nối chuỗi (dùng || thay vì CONCAT cho nhiều tham số)
SELECT "province_name" || ' - ' || "ward_name" AS full_address FROM "citizen_postbox";

-- SUBSTR
SELECT SUBSTR("code", 1, 8) AS short_code FROM "business_postbox";

-- INSTR — tìm vị trí chuỗi con
SELECT INSTR("email", '@') FROM "users";

-- REGEXP_LIKE — regex matching
SELECT * FROM "users" WHERE REGEXP_LIKE("email", '^[a-z]+@');

-- TRIM / LTRIM / RTRIM
SELECT TRIM("username") FROM "users";
```

### Xử lý ngày giờ

```sql
-- Thời gian hiện tại
SELECT CURRENT_TIMESTAMP FROM DUAL;  -- TIMESTAMP với timezone
SELECT SYSDATE FROM DUAL;            -- DATE (giờ server)
SELECT SYSTIMESTAMP FROM DUAL;       -- TIMESTAMP với timezone server

-- Trích xuất thành phần
SELECT EXTRACT(YEAR FROM "created_at") AS year,
       EXTRACT(MONTH FROM "created_at") AS month
FROM "users";

-- Cộng trừ ngày
SELECT "created_at" + INTERVAL '7' DAY AS next_week FROM "users";
SELECT "created_at" + INTERVAL '1' MONTH AS next_month FROM "users";

-- Format ngày
SELECT TO_CHAR("created_at", 'DD/MM/YYYY HH24:MI:SS') AS formatted FROM "users";

-- Parse chuỗi thành ngày
SELECT TO_DATE('15/03/2026', 'DD/MM/YYYY') FROM DUAL;
SELECT TO_TIMESTAMP('15/03/2026 10:30:00', 'DD/MM/YYYY HH24:MI:SS') FROM DUAL;

-- Khoảng cách ngày
SELECT TRUNC(SYSDATE) - TRUNC("created_at") AS days_since_created FROM "users";
```

### Conditional — CASE / DECODE

```sql
-- CASE (chuẩn SQL — khuyến nghị)
SELECT "username",
    CASE "status"
        WHEN 'ACTIVE'   THEN 'Hoạt động'
        WHEN 'INACTIVE' THEN 'Tạm khóa'
        ELSE 'Không xác định'
    END AS status_label
FROM "users";

-- DECODE (Oracle-specific — ngắn gọn hơn cho so sánh đơn giản)
SELECT "username",
    DECODE("status", 'ACTIVE', 'Hoạt động', 'INACTIVE', 'Tạm khóa', 'Không xác định') AS status_label
FROM "users";
```

## 7. Oracle JSON Support (12c+)

### Lưu JSON trong CLOB

```sql
CREATE TABLE "label_auto_assign_rules" (
    "id"          NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    "conditions"  CLOB CHECK ("conditions" IS JSON),
    ...
);

-- Insert JSON
INSERT INTO "label_auto_assign_rules" ("name", "conditions")
VALUES ('Rule 1', '{"conditions": [{"field": "subject", "operator": "CONTAINS", "value": "urgent"}]}');

-- Query JSON (Oracle 12c+)
SELECT "name", JSON_VALUE("conditions", '$.conditions[0].field') AS first_field
FROM "label_auto_assign_rules"
WHERE JSON_EXISTS("conditions", '$.conditions[*]?(@.field == "subject")');
```

### JPA mapping JSON

```java
@Column(name = "conditions", columnDefinition = "CLOB")
@JdbcTypeCode(SqlTypes.JSON)
private String conditions;

// Hoặc với custom object
@Column(name = "business_criteria", columnDefinition = "CLOB")
@JdbcTypeCode(SqlTypes.JSON)
private BusinessFilterCriteria businessFilterCriteria;
```

## 8. MERGE — Upsert Pattern

```sql
-- MERGE = INSERT hoặc UPDATE
MERGE INTO "label_display_config" target
USING (SELECT :propertyId AS property_id FROM DUAL) source
ON (target."property_id" = source.property_id)
WHEN MATCHED THEN
    UPDATE SET
        "display_type" = :displayType,
        "position" = :position,
        "updated_at" = CURRENT_TIMESTAMP,
        "updated_by" = :userId
WHEN NOT MATCHED THEN
    INSERT ("display_type", "position", "property_id", "created_by")
    VALUES (:displayType, :position, :propertyId, :userId);
```

## Checklist trước khi commit

- [ ] Dùng `VARCHAR2` (KHÔNG dùng `VARCHAR`)
- [ ] Boolean dùng `NUMBER(1)` (0/1)
- [ ] Tất cả FK columns có index
- [ ] Có `COMMENT ON TABLE` và `COMMENT ON COLUMN` cho bảng/cột quan trọng
- [ ] Soft delete dùng `"deleted" NUMBER(1) DEFAULT 0`
- [ ] Audit fields đầy đủ (theo `PropertyV2Auditable`)
- [ ] Phân trang dùng `FETCH FIRST ... ROWS ONLY` (Oracle 12c+)
- [ ] Constraint naming conventions tuân thủ: `pk_`, `fk_`, `uk_`, `ck_`, `idx_`
